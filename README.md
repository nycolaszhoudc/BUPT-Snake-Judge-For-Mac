# Snake AI Web Judge (贪吃蛇 AI 可视化评测系统)

这是一个用于开发和调试贪吃蛇 AI 的本地 Web 评测框架。通过提供可视化回放和自动化跑分功能，帮助开发者更直观地定位 AI 的逻辑问题并评估算法性能。系统主要支持 macOS 和 Linux。

## 主要功能

- **集成开发环境**：内置 Web 代码编辑器，支持 C 语言语法高亮，可在浏览器中直接修改 AI 源码并一键编译运行。
- **可视化回放**：记录 AI 每一步的操作，支持暂停、倍速播放和逐帧步进，便于复盘错误路径和死因。
- **自动化跑分**：模拟计分规则，支持 N=1 到 N=512 的批量压力测试，并自动折算为标准等级分。
- **多梯度地图**：提供 8 种测试场景，涵盖无障碍空地、零散障碍、连续墙壁以及混合地形等地图。
- **报告分享**：支持将源码、成绩和录像帧打包导出为 `.json` 文件，可通过导入文件在本地复现测试过程。

## 快速启动

### 1. 环境依赖

- 操作系统: macOS 或 Linux&#x20;
- 编译器: gcc 或 clang
- Python 环境: Python 3.x

### 2. 编译评测机

进入项目目录并编译底层的 C 语言评测机：

```bash
gcc snake_judge.c -o snake_judge
```

### 3. 启动 Web 服务

安装依赖并启动 Flask 服务：

```bash
pip3 install -r requirements.txt
python3 web_app.py
```

服务启动后，在浏览器访问 `http://127.0.0.1:5050` 即可开始使用。

> **常见问题排查**：如果在执行 `python3` 或 `pip3` 时系统提示找不到命令，可能是你的 macOS/Linux 环境默认绑定的是 `python` / `pip`（常见于通过 conda、pyenv 管理的环境，或某些精简版 Linux 发行版）。你可以尝试将命令替换为 `python -m pip install -r requirements.txt` 和 `python web_app.py`。

## AI 通信协议规范

AI 程序需要通过标准输入输出 (stdin/stdout) 与评测机进行交互。

### 回合输入

每回合评测机会通过 stdin 传入以下数据：

1. **20 行地图数据**：包含 `#`(墙壁), `.`(空地), `H`(蛇头), `B`(蛇身), `F`(食物), `O`(障碍物)。
2. **整数 N**：表示蛇每走 N 步长长一节。
3. **食物坐标**：`Y X`（先行后列）。若无食物，则传入 `100 100`。

### 回合输出

AI 需要在 stdout 输出一行数据，包含两个以空格分隔的值：

1. **移动方向**：`W` (上), `A` (左), `S` (下), `D` (右)。
2. **当前得分**：AI 自身计算的得分（用于校验）。

### 游戏结束处理 (重要)

当接收到特殊的食物坐标 `100 100` 时，表示游戏已结束（蛇死亡或通关）。此时 AI 必须：

1. 输出 20 行字符串，表示游戏结束时的最终地图状态。
2. 在第 21 行输出一个整数，表示最终得分。
3. **立刻调用** **`exit(0)`** **主动退出进程**，否则会导致评测机死锁超时。

### 评测限制

- **时间限制**：单步 400ms
- **内存限制**：64MB
- **代码长度限制**：24KB
- **栈限制**：8192KB
- **步数限制**：无&#x20;

## CLI 终端测试脚本

除了 Web 界面外，项目还提供了供进阶使用的命令行测试脚本：

- 单体快速验证：`python3 run_oj_tests.py ./my_ai`
- 多版本 AI 跑分排行：`python3 benchmark_all.py ./ai_v1 ./ai_v2`
- 批量极限抗压测试：`python3 batch_benchmark.py ./ai_v1 ./ai_v2 100`
